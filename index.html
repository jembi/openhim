OpenHIM

OpenHIM is a open source Health Information Mediator built to facilitate interoperability between disparate Health Information Systems.

Download this project as a .zip file Download this project as a tar.gz file

The Open Health Information Mediator (OpenHIM) is a middleware system based on an ESB architecture that aims to enable easier interoperability between disparate Health Information Systems.

Our vision to build a community of implementers and developers that support this work so that we can ease the burden of implementing interoperability and help enable clinical information sharing between clinicians in real world projects.

The reference implementation of this is for the Rwandan Health Information Exchange (RHIE). This project is know as the RHEA (Rwandan Health Enterprise Architecture) project. Please see the RHEA wiki for more details about the current reference implementation and about the RHEA project in general:

  • Health Information Mediator Documentation
  • RHEA Implementation wiki

Get involved!

If you would like to ask some questions or hold some discussions around the OpenHIM project please feel free to contact us on the OpenHIM Implementers mailing list. This group will be for people wanting to implement the OpenHIM software for thier own use case and for discussion of how you can start using this project now. Currently the software to pretty specific to the Rwandan use case but it is designed to be more generally applicable and that will be a major focal point for development going forward.

If you would like to get involved with this project, more technical developer discussions will take place on the OpenHIM Developer mailing list. Also, please visit us on github (https://github.com/jembi/openhim) to find out how to get setup and test out the project.

For further development of the OpenHIM we will hold discussion on the OpenHIM Developers mailing list, so sign up now if you are interested in keeping up to date.

We hope to hear from you soon!

Features

  • Exposes a single RESTful interface to all infrastructure services
  • Ensures communicate between the client and the OpenHIM is secure
  • Provides a platform for transaction orchestration
  • Provides a platform for transaction translation
  • Logs all transaction details and metadata

About
